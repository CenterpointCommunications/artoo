ARDrone

Driver

Allows user to send flight control commands to an ARDrone. Flight control commands are those used to tell the ARDrone to take off, land, and or any number of other autonomous flight manuvers. Communication with the ARDrone takes place using a WiFi connection.

How to establish a connection and setup the driver

require 'artoo'
connection :ardrone, :adaptor => :ardrone, :port => '192.168.1.1:5556'
device :drone, :driver => :ardrone, :connection => :ardrone

Public instance methods:

start → drone

Starts up the drone.

stop → drone

Stops the drone.

hover → drone

Causes the drone to hover at current altitude.

land → drone

Causes the drone to begin its landing sequence.

take_off → drone

Causes the drone to take off, if already started.

emergency → drone

Causes the drone to make an immediate emergency landing

front_camera → drone

Causes the drone to turn on the front-facing camera.

bottom_camera → drone

Causes the drone to turn on the bottom-facing camera.

up → drone

Causes the drone to increase altitude.

down → drone

Causes the drone to decrease altitude.

left → drone

Causes the drone to bank to the left.

right → drone

Causes the drone to bank to the right.

forward → drone

Causes the drone to bank forward.

backward → drone

Causes the drone to bank backward.

turn_left → drone

Causes the drone to rotate to the left.

turn_right → drone

Causes the drone to rotate to the right.

Example of controlling an ARDrone.

require 'artoo'

connection :ardrone, :adaptor => :ardrone, :port => '192.168.1.1:5556'
device :drone, :driver => :ardrone, :connection => :ardrone

work do
  drone.start
  drone.take_off

  after(15.seconds) { drone.hover.land }
  after(20.seconds) { drone.stop }
end
